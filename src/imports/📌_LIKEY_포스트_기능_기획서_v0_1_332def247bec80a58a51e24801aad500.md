# **1. 문서 개요**

### **목적**

본 문서는 LIKEY 서비스 내 포스트 생성 및 포스트 조회 기능을 구현하기 위한 최소 기능(v0.1)을 정의하는 것을 목적으로 한다.

### **범위**

- 포스트 생성 (Post Creation)
- 포스트 조회 (Post View)

### **제외 범위**

- 임시 저장 / 예약 게시
- 공유 (링크, X 등 외부 채널)
- 신고 및 크리에이터의 댓글 관리 기능

# 2. 용어 및 사용자 정의

### **2.1 용어 정의**

| **용어** | **정의** |
| --- | --- |
| 포스트 | 크리에이터가 작성하여 서비스 내에 게시하는 단위 콘텐츠 |

| 개별 구매 | 멤버십 가입 여부와 무관하게 특정 포스트 단위로 결제하여 잠금된 미디어를 조회하는 방식 |

| 잠금 해제 **CTA** | 잠금 상태 포스트에서 사용자가 추가 권한을 획득할 수 있도록 제공되는 행동 유도 요소. 멤버십 구독 또는 개별 구매 진입 수단으로 사용된다. |
| 반응 수 | 포스트에 표시되는 좋아요 수와 댓글 수를 의미한다. |

# **3. 포스트 생성 (Post Creation)**

## **3.1 기능 목적**

크리에이터가 미디어 기반 콘텐츠를 업로드하고, 조회 권한 및 판매 조건을 설정하여 게시할 수 있도록 한다.

---

## **3.2 유저 플로우**

![image.png](%F0%9F%93%8C%20LIKEY%20%ED%8F%AC%EC%8A%A4%ED%8A%B8%20%EA%B8%B0%EB%8A%A5%20%EA%B8%B0%ED%9A%8D%EC%84%9C%20v0%201/image.png)

<aside>

1. 크리에이터가 포스트 작성 화면에 진입한다.
2. 사진 및/또는 영상을 첨부한다.
3. 시스템은 파일 규격을 검증하고, 실패 시 에러 메시지를 노출한다.
4. 검증이 완료된 미디어는 업로드되며, 업로드 완료 후 본문 입력 단계로 이동한다.
5. 크리에이터는 본문을 선택적으로 입력한다.
6. 공개 범위를 선택한다.
7. 멤버십 공개를 선택한 경우, 노출 대상이 되는 멤버십 티어를 선택한다.
8. 개별 구매를 ON으로 설정한 경우, 하트 기준 가격을 입력한다.
9. 게시 버튼 클릭 시 시스템은 필수 입력값을 검증한다.
10. 검증 성공 시 포스트 생성 요청을 수행하고, 성공하면 피드 반영 후 포스트 상세 화면으로 이동한다.
11. 검증 또는 생성 실패 시 에러/오류 메시지를 노출한다.
</aside>

## **3.4 예외 처리**

- **중복 게시 요청**
    
    사용자가 게시 버튼을 반복 클릭하더라도 포스트 생성 요청은 1회만 처리한다.
    
- **비정상적 대량 게시 요청**
    
    짧은 시간 내 과도한 포스트 생성 요청이 발생하는 경우, 서비스 안정성과 어뷰징 방지를 위해 생성 요청을 제한한다. 
    
    - v0.1에서는 **분당 3건 이상 또는 10분 내 10건 이상 게시 요청 시** 추가 생성 요청을 제한한다.
- **작성 중 화면 이탈**
    
    작성 또는 업로드 도중 사용자가 화면을 이탈할 경우, 저장되지 않은 입력값은 유지되지 않음을 안내한다.
    
- **네트워크 및 서버 오류**
    
    업로드 또는 포스트 생성 요청 처리 중 네트워크/서버 오류가 발생한 경우, 실패 메시지를 노출하고 재시도를 유도한다.
    
- **멤버십 티어 유효성 변경**
    
    게시 시점에 선택한 멤버십 티어가 더 이상 유효하지 않은 경우, 게시 요청을 제한하고 최신 설정 기준으로 재선택을 유도한다.

# **4. 포스트 조회 (Post View)**

## **4.1 기능 목적**

사용자가 피드 및 포스트 상세 화면에서 포스트를 조회하고, 권한에 따라 콘텐츠를 소비하며, 필요 시 멤버십 구독 또는 개별 구매를 통해 잠금된 미디어에 접근할 수 있도록 한다.

---

## **4.2 유저 플로우**

![image.png](%F0%9F%93%8C%20LIKEY%20%ED%8F%AC%EC%8A%A4%ED%8A%B8%20%EA%B8%B0%EB%8A%A5%20%EA%B8%B0%ED%9A%8D%EC%84%9C%20v0%201/image%201.png)

<aside>

1. 사용자는 피드 영역에서 포스트 리스트를 조회한다.
2. 사용자는 포스트의 노출 상태에 따라 본문, 미디어, 라벨, 반응 수를 확인한다.
3. 사용자는 잠금된 포스트에서 잠금 해제 CTA를 확인하고, 필요 시 구독 또는 구매를 선택한다.
4. 사용자는 특정 포스트를 선택해 상세 화면으로 진입한다.
5. 사용자는 상세 화면에서 포스트 콘텐츠와 댓글을 확인한다.
6. 사용자는 좋아요를 누르거나 댓글을 작성하며 상호작용한다.
</aside>

## 4.5 예외처리

- **삭제되었거나 더 이상 제공되지 않는 포스트 접근**
    
    사용자가 삭제되었거나 더 이상 제공되지 않는 포스트에 접근한 경우, 포스트 내용을 노출하지 않고 접근 불가 메시지를 표시한다.
    
- **권한 획득 후 상태 반영 지연**
    
    구독 또는 구매가 완료되었으나 포스트 상태 반영이 지연되는 경우, 동일 포스트 재조회 시 갱신된 상태를 반영한다.
    
- **비정상적 상세 진입 반복**
    
    짧은 시간 내 동일 포스트 또는 다수 포스트의 상세 화면 진입을 과도하게 반복하는 경우, 서비스 안정성을 위해 추가 조회 요청을 제한한다.
    
    ◦ 동일 포스트 10초 내 5회 이상, 또는 1분 내 20회 이상 상세 진입 요청 시 추가 요청을 제한한다.
    
- **잠금 해제 CTA 중복 요청**
    
    사용자가 잠금 해제 CTA를 반복 클릭하더라도 동일 요청은 1회만 처리한다.
    
- **좋아요/댓글 중복 요청**
    
    사용자가 좋아요 토글 또는 댓글 작성/삭제를 매우 짧은 시간 안에 반복 수행하는 경우, 중복 요청을 제한한다.
    
    ◦ 좋아요 2초 내 3회 이상, 댓글 작성 5초 내 3회 이상 요청 시 추가 요청을 제한한다.
    
- **네트워크 및 서버 오류**
    
    포스트 조회, 상세 진입, 댓글 조회, 권한 획득 과정에서 네트워크 또는 서버 오류가 발생한 경우, 오류 메시지를 표시하고 재시도를 유도한다.
